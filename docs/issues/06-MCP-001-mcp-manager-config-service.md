# 🧾 MCP-001 MCP Manager 設定編集サービス

## 🎯 背景/目的

- MCP Manager View の追加、編集、削除を安全に実装するため、`config.toml` の MCP ブロック操作をサービス化する。

## 📌 要件

- 主要項目のみを編集対象とする。
- `env` は Key-Value のフォーム項目として編集対象に含める。
- フォーム未対応項目は保存時に削除しない。
- `enabled_tools` と `disabled_tools` は同時設定不可とする。

## 🛠️ スコープ / 作業内容

- MCP ブロックの読み取り、更新、追加、削除サービスを追加する。
- stdio / http の主要項目をフォームモデルへ変換する。
- `[mcp_servers.<id>.env]` をフォームモデルへ変換し、保存時に再構築する。
- 保存時バリデーションを実装する。
- 既存 MCP Explore の ON/OFF トグル処理を再利用できる形に整える。
- 関連テストを追加する。

## ✅ AC（受け入れ基準）

- [x] [機能] 主要項目を読み取り、stdio / http ごとのフォームモデルへ変換できる。
- [x] [機能] `env` を Key-Value 一覧として読み取り、保存時に `[mcp_servers.<id>.env]` ブロックで更新できる。
- [x] [機能] 保存時に不要な `url` または `command` / `args` を削除できる。
- [x] [状態/エラー] サーバー名重複、必須項目不足、負数 timeout、tools 同時指定、`env` のキー検証を実施できる。
- [x] [機能] フォーム未対応項目は保存後も保持される。
- [x] [テスト] 読み取り、保存、追加、削除、バリデーションをテストする。
